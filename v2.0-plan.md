# NFL Predictor — v2.0 Plan

## Overview

v2.0 adds a dedicated **Player Stats** section to the app alongside the existing Make Predictions, View Standings, and Playoff Seeding tabs. This section is largely self-contained and does not interact with prediction state — it's a standalone player browser and stat viewer powered by live data from the ESPN unofficial API.

## Features

### Player Stats Tab (v2.0 Core)

A new top-level tab in the nav bar. The entry point is a team-based browser — 32 team cards, each expanding to show the roster. A search bar at the top allows jumping directly to any player by name.

**Player list / team browser:**
- 32 team cards (grouped by conference/division)
- Each card expands to show the roster for that team
- Global search bar filters across all players in real time
- Position filter chips: All | QB | RB | WR | TE | OL | DL | LB | DB | K/P
- Each player entry shows: headshot thumbnail, name, number, position

**Player profile (full-page view):**

Clicking a player pushes a full-page profile. A back button returns to the browser.

- **Headshot** prominently featured (large, top of profile)

- **Mobile:** headshot centered above info on narrow screens; accordion works identically

## API

### Primary: ESPN Unofficial API

**Base URL:** `https://site.api.espn.com/apis/site/v2/sports/football/nfl/`

| Endpoint | Use |
|---|---|
| `/teams/{teamId}/roster` | Load team roster |
| `/athletes/{playerId}` | Player bio, position, experience |
| `/athletes/{playerId}/stats` | Season and career stats |

**Headshots:** `https://a.espncdn.com/i/headshots/nfl/players/full/{espn_player_id}.png`

**Pros:** Free, no auth, CORS-friendly, includes headshots, widely stable.
**Cons:** Unofficial/undocumented — could change without notice.

### Secondary: Sleeper API

**Base URL:** `https://api.sleeper.app/v1/`

Used as a fallback for stats if ESPN is unavailable. Not the primary source — no headshots, large initial payload (~6MB for all players), fantasy-focused.

### Grades / Ratings

- **QBs:** ESPN QBR (available free via ESPN API — a legitimate advanced metric)
- **All other positions:** App-computed efficiency metrics derived from available stats (yards per carry, completion %, tackle rate, etc.). Labeled clearly as "App Rating" or similar — never implied to be PFF.
- **PFF grades are not available for free.** PFF is a paid service with no public API. Revisit if that changes.

## Caching Strategy

### Client-Side (v2.0)

Thin cache layer in `/utils/playerCache.js` — wraps `fetch`, stores responses in `localStorage` with a timestamp. Checks freshness before hitting the network.

| Data | TTL | Reason |
|---|---|---|
| Team roster | 24 hours | Changes weekly at most |
| Player bio/metadata | 24 hours | Rarely changes |
| Current season stats | 1 hour | Updates after each game day |
| Prior season stats | Indefinite | Static once season ends |

**Pros:** Zero infrastructure, works with existing Docker setup.
**Cons:** Per-browser only, ~5–10MB localStorage budget.

The fetch logic will be written to swap to a local proxy URL with a one-line change, so upgrading later is painless.

### Server-Side Proxy (Deferred)

If the app sees significant traffic or ESPN proves unreliable, add a lightweight proxy container (Node/Express or nginx `proxy_pass`) to the Docker Compose stack. Centralizes caching, eliminates CORS entirely, enables ESPN → Sleeper fallback logic server-side.
