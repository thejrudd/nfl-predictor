# Token Optimization Plan

**Goal:** Reduce per-session token consumption in Claude Code and Codex without losing useful context.  
**Drafted:** 2026-04-25

---

## Background

Claude Code auto-loads `CLAUDE.md` on every session. The "Docs First" section in CLAUDE.md signals which docs are canonical, implicitly expanding the effective context window. Additionally, large generated data files and oversized component files get pulled in during file-targeted work, compounding token cost. This plan addresses all three layers.

---

## Priority 1 — Exclude Generated Data Files (Very High Impact, Trivial Effort)

**Problem:** Two auto-generated data files total ~12,100 lines (≈245 KB) and are pulled into context any time a file in `src/data/` is referenced:

| File | Lines |
|---|---|
| `src/data/rookieProduction.generated.js` | 8,879 |
| `src/data/rookieGameLogs.generated.js` | 3,223 |

These files are never edited — they're regenerated by scripts. Loading them wastes massive context.

**Fix:** Create a `.claudeignore` at project root:

```
src/data/rookieProduction.generated.js
src/data/rookieGameLogs.generated.js
```

This tells Claude Code to skip these paths during glob/context expansion while still allowing direct reads when explicitly needed.

**Savings:** ~12,100 lines per context load avoided.

---

## Priority 2 — Archive Stale or Future-Only Docs (High Impact, Low Effort)

**Problem:** 7 docs in `docs/` describe either completed proposals, experimental features, or future roadmap items. They inflate CLAUDE.md's "Docs First" surface area and can be pulled into context unnecessarily.

**Candidates for `docs/archive/`:**

| File | Reason |
|---|---|
| `Shell Redesign Proposal.md` | Completed proposal — shipped, no longer actionable |
| `AP Action Photos Integration.md` | Experimental integration — not in active build |
| `Ollama Delegation.md` | Experimental / speculative feature |
| `Authentication And Memberships.md` | Future roadmap only |
| `Pretext Integration.md` | Future / speculative |
| `Open Pencil Integration.md` | Future / speculative |
| `Card Glow Effect.md` | One-off visual note, not active feature doc |

**Fix:**
1. Create `docs/archive/` directory.
2. Move the 7 files listed above into it.
3. Remove their references from `docs/Home.md` (add a single "Archive" link instead).
4. Do **not** remove references from CLAUDE.md — the "Docs First" section only lists the 8 active docs, so no change is needed there.

**Savings:** ~760 lines removed from browseable doc surface.

## Priority 3 — Trim CLAUDE.md Redundancies (Medium Impact, Low Effort)

**Problem:** CLAUDE.md has two sections that duplicate dedicated docs:

- **"Design System — Broadcast Editorial"** (lines 37–67): repeats content from `docs/Design Tokens.md`. The key rules are useful as a quick-ref, but the team color palette subsections and token table details add ~20 lines of redundancy.
- **"Navigation Architecture"** (lines 54–77): breakpoint values and CSS snippets echo `docs/Architecture Map.md`.

**Fix:**
- Collapse "Design System" to a 4-line quick-ref (signature color warning, font-size rule, safe area, motion easing). Remove team color subsections — they're already in CLAUDE.md as a 2-line reference to the source files.
- Collapse "Navigation Architecture" to the state variable list only. Remove the CSS snippet — it belongs in `docs/Architecture Map.md`.

**Savings:** ~30–40 lines from CLAUDE.md (auto-loaded every session).

## Priority 4 — Refactor CompanionTrade.jsx (High Impact, High Effort)

**Problem:** `src/components/CompanionTrade.jsx` is 5,291 lines with 64 imports and 27 state hooks. Any trade-related task forces this entire file into context.

**Proposed split:**

| New File | Responsibility | Est. Lines |
|---|---|---|
| `CompanionTradeSetup.jsx` | Player selection, filter UI, roster display | ~1,500 |
| `CompanionTradeProposal.jsx` | Proposal cards, scoring breakdown, package comparison | ~2,000 |
| `CompanionTradeAnalytics.jsx` | Upgrade score, KTC delta, explanation cards | ~1,500 |
| `CompanionTrade.jsx` (shell) | Orchestration, shared state, routing between views | ~300 |

**Savings per targeted task:** Instead of loading 5,291 lines, Claude loads only the relevant 1,500–2,000-line module.

**Note:** This is a significant refactor. Coordinate with active feature work before starting.

## Priority 5 — Extract Shared Modal Wrapper (Medium Impact, Medium Effort)

**Problem:** 13 modal/ActionSheet components copy-paste the same backdrop + scroll-lock + container pattern (documented in CLAUDE.md). Each file carries the full implementation.

**Fix:** Create `src/components/Modal.jsx` — a thin wrapper that handles:
- `fixed inset-0 z-50 flex items-center justify-center` backdrop
- `useBodyScrollLock` on mount
- Backdrop click → `onClose`
- `rounded-2xl` container with `mx-4` and configurable `maxWidth`

All 13 existing modals can then replace ~15 lines of boilerplate with `<Modal onClose={onClose} maxWidth="...">`.

**Savings:** ~300 lines removed across the codebase; CLAUDE.md's Modal Pattern section can shrink to a 1-line reference.

---

## Priority 6 — Migrate Code Comments to Docs (Low Impact, Low Effort)

**Problem:** ~1,375 comment lines across `src/`. Several high-complexity files are comment-heavy:

- `src/context/SleeperContext.jsx` — implementation notes embedded inline
- `src/utils/tradeEngine.js` — explanation comments that belong in `docs/Trade Engine.md`
- `src/components/ShareableImage.jsx` — single-feature file with multi-paragraph comment blocks

**Fix:** For each file, migrate implementation-rationale comments to the appropriate doc. Keep only non-obvious constraint comments in code (per CLAUDE.md style rules).

**Savings:** ~500–800 lines; each line removed is a line not loaded when the file enters context.
